Comando para crear app

Crear Entorno virtual: py -m venv .\[nombre de la carpeta]\
Dar permisos varios: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process
Activar el entorno virtual: Ir a la carpeta SCRIPT y ejecutar el file .\activate

- crear carpta project
-cd project
- crear carpeta config con comando: django-admin startproject config .
- configurar lenguaje y zona horaria en settings.py

-crear app general (core) en project: django-admin startapp core
- crear cliente y producto
- registrar las apps en el settings de la carpta config en INSTALLED_APPS: 'core', 'cliente', 'producto'
- crear dentro de core: templates/core/index.html, repetir lo mismo en las otras apps (se crean sub carpetas con el misno nombre para evitar anbiguedades al momento de buscar los html)

-copiar el urls.py de config en core
-crear funcion que renderize la vista home en el views de core
-en el urls de core llamar a la vista que de devuelve en la ruta raiz
- incluir el core urls en el urls de config
- levantar el servidor con python manage.py runserver desde el directorio raÃ­z del proyecto
- hacer que hore de core se ejecuta al inicio del servidar

*
si subimos el proyecto a github podemos crear el archivo requirements.txt con las dependencias en la raiz del proyecto

-usando pip freezze >> requirements.txt
-al clonarlo podemos instalar esas dependencias en el entorno virtual usando pip install -r requirements.txt
*

. para navegar entre las url tenemos que establer un nombre dentro de cada archivo urls utilizando x ejemplo app_name = 'cliente'
- tambien tenemos que pasarle el parametro name a cada path 

===
app_name = 'cliente'

urlpatterns = [
    path('',views.home, name='index' ),
]

===

- crear migraciones y pasarlas a la bd


python manage.py makemigrations
python manage.py migrate


para poder trabajar con los modelos hay que registrarlos en el admin.py

admin.site.register(models.Pais)
admin.site.register(models.Cliente)

crear un super usuario

python manage.py createsuperuser
